## Design Context

### Users

**Audiência universal, funil de valor nos experientes.**

O FinHub serve todos os níveis — desde quem nunca investiu até ao investidor activo com portfolio diversificado. A estratégia de aquisição é via conteúdo educativo (artigos, cursos, vídeos) que captura utilizadores iniciantes. O retorno e a retenção vêm dos utilizadores mais experientes, que usam ferramentas avançadas (stocks, REITs, FIRE simulator, mercados), consomem creators especializados e interagem na comunidade.

**Implicação de design:** a interface tem de funcionar em dois modos mentais simultaneamente — superfície limpa e convidativa para iniciantes, densidade e profundidade acessíveis para power users. A complexidade deve ser progressiva, nunca forçada.

**Contexto de uso:** maioritariamente mobile para conteúdo e comunidade (feed, artigos, creators). Desktop para ferramentas e análise de mercados. SSR/SEO crítico — muitos utilizadores chegam via pesquisa orgânica.

### Brand Personality

**Sério. Moderno. Educativo.**

- **Sério**: merece confiança. Finanças são importantes. O design não deve trivializar dinheiro ou investimento. Sem gimmicks, sem cores gritantes, sem urgência artificial.
- **Moderno**: fluido, rápido, sem fricção. Sente-se contemporâneo — não um banco de 2010 nem uma fintech genérica.
- **Educativo**: convida à descoberta. Cada ecrã pode ensinar algo. O design suporta aprendizagem, não apenas consulta.

**Tom visual**: confiante sem ser arrogante. Acessível sem ser infantil. Denso onde tem de ser, mas nunca sufocante.

### Aesthetic Direction

**Referências por área:**

| Área | Referência | O quê especificamente |
|------|-----------|----------------------|
| Descoberta de conteúdo | Netflix | Hero banner, content rows horizontais, scroll com snap, hierarchy visual forte |
| Ferramentas & Mercados | Trading 212 | Dados apresentados com elegância, UI limpa para informação financeira, não intimidante |
| Comunidade & Creators | Instagram + Discord | Feed social, cards de creator, sentido de comunidade activa, notificações |
| Geral | — | Fluido, animações subtis, transições rápidas |

**Anti-referências explícitas:**
- **Finviz**: tabelas densas, dados sem hierarquia, visualmente agressivo — nunca assim
- **SeekingAlpha**: demasiado text-heavy, datado, não mobile-friendly — nunca assim
- Nada "só números" — dados financeiros sempre contextualizados com hierarquia clara
- Sem vermelho/verde a piscar, sem urgência de trading agressivo

**Tema:** dark mode é feature de primeira classe (não afterthought). Light mode é o default de entrada.

**Paleta em uso:**
- Primary: azul (hsl 222 47%) — confiança, profissionalismo
- Success/Bull: verde (#22C55E) — crescimento, positivo
- Danger/Bear: vermelho (#EF4444) — risco, negativo (usar com contenção)
- Neutral: slate/cinza frio — backgrounds, texto secundário
- Market: `market-bull`, `market-bear`, `market-neutral` tokens definidos

**Tipografia:** Inter exclusivamente (a rever — heading diferenciada em avaliação). Tabular nums obrigatório para dados financeiros (`font-variant-numeric: tabular-nums`).

**⚠️ Gap crítico identificado (critique 2026-03):** O `--primary` actual é `222.2 47% 11%` — praticamente preto. Não existe cor de acento real. Todos os CTAs são botões escuros sem energia visual. Este é o maior gap de identidade da plataforma actualmente.

### Design Principles

**1. Clareza sobre completude**
Nunca mostrar todos os dados ao mesmo tempo. Surfaçar o que importa, esconder complexidade atrás de progressive disclosure. Se um ecrã parece um dashboard de aviação, está errado.

**2. Conteúdo em primeiro lugar**
Cada ecrã deve convidar à exploração, não sobrecarregar com informação. A estrutura Netflix (hero + rows) é o modelo mental base para discovery. Aplicar em conteúdo, creators, comunidade.

**3. Confiança por contenção**
Dados financeiros exigem respeito. Whitespace e tipografia comunicam confiança melhor do que decoração. Vermelho e verde existem para significar — não para decorar. Usar com intenção e parcimónia.

**4. Comunidade como produto**
A camada social (creators, feed, comunidade) é tão importante quanto as ferramentas financeiras. Design para ligação humana ao lado de dados. Cards de creator, avatares, interacções sociais merecem o mesmo cuidado que gráficos de stocks.

**5. Profundidade progressiva**
Utilizadores iniciantes vêem uma superfície limpa e convidativa. Power users desbloqueiam densidade. Nunca forçar complexidade em quem não está pronto. A mesma feature pode ter camadas — resumo visível, detalhe em expandir/drill-down.

## Tech Constraints para Design

- **Tailwind CSS 3** — usar tokens definidos em `design-tokens.css`, não hardcoded values
- **shadcn/Radix UI** — componentes base já definidos, estender via `className`, não sobrescrever
- **Dark mode via `.dark` class** (next-themes) — qualquer componente novo deve suportar ambos os temas
- **SSR (Vike)** — evitar lógica de UI que dependa exclusivamente de `window`/`document` sem guards
- **Inter via @fontsource** — fonte já carregada, não adicionar Google Fonts
- **Recharts** — para gráficos financeiros, usar `chart-1` a `chart-5` tokens + `market-bull`/`market-bear`
- **Tabular nums** — obrigatório para qualquer número financeiro (preços, percentagens, valores de portfolio)
